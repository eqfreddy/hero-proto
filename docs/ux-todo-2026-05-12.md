## 🔥 P0 — Critical UX bugs (fix this session)

- [ ] **Setup → Fight! still routes to legacy `battle-arena.html`, NOT the new Quaternius 3D scene.** The new viewer (`BattlePlayRoute` + `Battle3DScene`) only runs for *interactive* battles created via the interactive session API. Normal stage Fight! hits `POST /battles` and gets force-redirected to the static replay page (CraftPix 2D). The user thought they were seeing KayKit; they're actually seeing CraftPix because the 3D scene isn't even loaded. **Decision needed:** either (a) add an "Interactive" toggle on `BattleSetupRoute` that creates an interactive session and routes to `/battle/:id/play`, or (b) wire ALL normal stage battles through the new viewer and retire `battle-arena.html`.
- [ ] **Home page (`/app/me`) renders bespoke chrome — no shared NavBar, no CurrencyBar.** Playwright confirmed `hasNavBar: false, hasCurrencyBar: false` on Home vs `true` on all 18 other pages. The TopBar is custom. Mobile users see NO currency display on the home dashboard. **Fix:** retire the bespoke TopBar in `MeRoute`; use the shared `Shell` chrome like every other page. Promote currency + energy + tickets to the persistent header.
- [ ] **Deep links into `/battle/setup` return JSON 404** — backend catch-all only fallback-serves SPA for `/app/*`. Anyone bookmarking a battle URL or refreshing during play hits the API 404 instead of the SPA. **Fix:** extend the FastAPI catch-all to serve `app/static/spa/index.html` for `/battle/*` too.
